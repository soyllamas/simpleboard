# General Code Style Conventions

- Keep implementation concise.
- Keep implementation short and simple.
- Keep implementation elegant.
- Keep implementation idiomatic.
- Design light-weight apis.
- Design intuitive apis.
- Use intention-revealing names.
- Make names pronounceable and searchable.
- Avoid mental mapping and abbreviations.
- Keep functions small.
- Functions should do one thing well.
- Maintain one level of abstraction per function.
- Avoid writing comments.
- Explain why, not what, in comments.
- Order code to read top-to-bottom naturally.
- Choose the simplest solution that works.
- Avoid unnecessary complexity and cleverness.
- Favor readability over brevity.
